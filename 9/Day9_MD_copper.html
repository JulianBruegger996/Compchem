  • Size: Tuple how often the unit cell will be repeated in x,y,z direction
  • pbc: set periodic boundary conditions to True

Reuse the function from the water MD to run your simulations. As an additional argument the temperature should be included. Additionally, the positions of the atoms should be saved only every 5th simulation step, otherwise the trajectory file will be too large. As thermostat we will use Langevin and the friction can be set to 0.05 for the equilibration run, and to 0.02 for the production run. The function will begin with an equilibration run, followed by a production run, both have the same length and stepsize. This needs to be done every time the temperature is changed, otherwise the system will not be equilibrated at the new temperature.

In [4]:
def run_simulation_NVT(dt, steps, temp, tag, atoms):
    # copy the atoms object to avoid modifying the original structure
    # attach the EMT calculator to the copied atoms object
    md_atoms = atoms.copy()
    md_atoms.calc = EMT()

    # Set the initial velocities corresponding to T=temp from Maxwell Boltzmannn distribution
    MaxwellBoltzmannDistribution(md_atoms, temperature_K=temp)

    print("Starting equilibration run...")
    # perform equilibration run
    # change the tag for the equilibration run to avoid overwriting the production run log and trajectory files
    # define the Langevin dynamics object with the copied atoms, time step, temperature, friction coefficient = 0.05, and log file name
    dyn = Langevin(md_atoms, dt, temperature_K=temp, friction=0.05, logfile=f'{tag}_equi.log')
    # attach a trajectory object to the dynamics object to save the trajectory every 5 steps (keyword: interval=5) using the tag for the equilibration run   
    # run the dynamics for the specified number of steps
    traj = Trajectory(f'{tag}_equi.traj', mode="w", atoms=md_atoms)
    dyn.attach(traj.write, interval=5)
    dyn.run(steps)

    print("Finished equilibration! Starting production run...")
    # perform production run
    # define the Langevin dynamics object with the copied atoms, time step, temperature, friction coefficient = 0.02, and log file name (tag) for the production run
    dyn = Langevin(md_atoms, dt, temperature_K=temp, friction=0.02, logfile=f'{tag}.log')
    # attach trajectory object to the dynamics object to save the trajectory every 5 steps (keyword: interval=5) using the tag for the production run
    # run the dynamics for the specified number of steps
    traj = Trajectory(f'{tag}.traj', mode="w", atoms=md_atoms)
    dyn.attach(traj.write, interval=5)
    dyn.run(steps)
    print("Finsished!")

def get_peaks(data, time_step, stride):
    peaks = []
    times = []
    previous = data[0]
    current = data[0]

    for i, next in enumerate(data):
        if current > previous and current > next:
            peaks.append(current)
            times.append((i-1) * stride * time_step)
        previous = current
        current = next

    return times, peaks
